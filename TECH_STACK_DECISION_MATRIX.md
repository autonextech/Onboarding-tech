# 🛠️ TECHNOLOGY STACK & ARCHITECTURE DECISION MATRIX

This document outlines the core technologies chosen for the Employee Onboarding Platform, complete with the rationale, cost analysis, and scaling constraints.

## 1. Frontend Architecture ⚛️
**Stack:** React 18 + Vite + Zustand + TanStack Query + Tailwind CSS

| Technology | Justification | Trade-offs |
|------------|---------------|-------------|
| **React 18** | Industry standard, massive ecosystem, Concurrent rendering for smooth UI transitions during heavy quiz loads. | Steeper learning curve than simpler templating languages. |
| **Vite** | Unparalleled HMR (Hot Module Replacement) speed. Drops local build times from 30s (Webpack) to <1s. | Slightly different ecosystem integration than standard Webpack. |
| **Zustand** | Zero-boilerplate global state. Perfect for managing User Auth state and minimal UI toggles. | Less native middleware than Redux. |
| **TanStack Query** | Handles caching, background syncing, and pagination for modules/tasks instantly. Essential for `<200ms` perceived latency. | Adds bundle size, requires strict cache-key conventions. |
| **Tailwind CSS** | Enables rapid iteration, ensures design consistency, and drops shipped CSS payload to `<10kb`. | Pollutes JSX markup slightly. |

## 2. Backend Architecture 🟢
**Stack:** Node.js + Express + PostgreSQL + Redis

| Technology | Justification | Trade-offs |
|------------|---------------|-------------|
| **Node.js + Express** | High concurrency via async I/O. Extremely fast for the many small JSON requests generated by platform quizzes and dashboard stats. | Single-threaded nature requires careful handling of CPU-bound tasks (e.g. heavy PDF generation). |
| **PostgreSQL** | Relational integrity is mandatory for financial/employee data. Handles complex joins (Members -> Mentors -> Modules -> Tasks) efficiently. | Harder to shard horizontally compared to NoSQL. |
| **Redis** | In-memory key-value store. Used for session management, rate-limiting, and caching heavily hit leaderboards or dashboard statistics. | Volatile by default; requires persistence config if used as primary source of truth (which we won't). |

## 3. DevOps, Cloud & Infrastructure ☁️
**Stack:** AWS, Docker, Kubernetes, GitHub Actions

| Technology | Justification | Trade-offs |
|------------|---------------|-------------|
| **Docker + K8s** | Containerization guarantees parity between dev and prod. K8s allows auto-scaling nodes during large onboarding batches (start of month). | Operational complexity for small teams. |
| **AWS AWS/S3/EC2** | S3 for immutable module assets (videos, PDFs). EC2/EKS for generic compute. CloudFront for edge-caching video delivery. | Vendor lock-in; potentially confusing IAM rules. |
| **GitHub Actions** | CI/CD tightly integrated with our codebase. Automates testing and deployment directly to AWS. | GitHub limits on free-tier compute minutes. |

## 4. Monitoring & Third-Party Integrations 📊
- **Datadog**: Used for APM (Application Performance Monitoring) to ensure the `<200ms` p95 API response time SLA is met.
- **Sentry**: Real-time frontend and backend error tracking. Captures stack traces automatically.
- **SendGrid**: Handles transaction emails (New assigned tasks, Mentor pairing notifications). Ensures high deliverability.

## 5. Cost Analysis at Scale 💵
Based on a projection of **10,000 active candidates and admins** per month:

- **AWS Managed DB (RDS Multi-AZ)**: ~$400/mo
- **EKS Compute Nodes (m5.large x4)**: ~$300/mo
- **AWS S3 Storage + Cloudfront Data Transfer (5TB)**: ~$450/mo
- **Redis (ElastiCache)**: ~$100/mo
- **Datadog/Sentry/Sendgrid Subscriptions**: ~$650/mo
---
**Total Estimated Opex:** ~$1,900 / month  
**Cost per user:** ~$0.13 / month  

*Conclusion: The stack offers enterprise-grade reliability and performance while remaining highly cost-effective and familiar to modern engineering talent.*
